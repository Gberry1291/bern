{% trans "How it works" %}

{% trans "A privacy-first search experience that helps families find disability-appropriate housing faster — using a blend of structured filters and AI understanding." %}

  • {% trans "Understands natural language (not just keywords)" %}
  • {% trans "Combines accessibility filters + semantic matching" %}
  • {% trans "Designed with privacy and data minimization from day one" %}

{% trans "The problem" %}

{% trans "Finding housing for people with disabilities is rarely a simple checkbox exercise. Families often search while under stress, providers describe similar services in different words, and important accessibility details can be buried in long descriptions. Our goal is to make searching feel human — and still remain precise." %}

{% trans "What happens when someone searches" %}

{% trans "1) We detect intent signals" %}

{% trans "Users can write naturally (for example: “My mother needs more care, she forgets a lot.”). The system extracts helpful signals such as age cues (when provided), accessibility needs, care intensity cues, and explicit condition keywords. This guides search without requiring medical forms." %}

{% trans "2) We apply hard filters for safety and accuracy" %}

{% trans "Some requirements aren’t optional. If a user needs wheelchair accessibility or searches for youth housing, results that don’t meet those criteria can be excluded immediately. This prevents irrelevant or unsafe recommendations." %}

  • {% trans "Language availability (DE/FR/IT/EN)" %}
  • {% trans "Age suitability (youth vs adult)" %}
  • {% trans "Accessibility flags when selected" %}

{% trans "3) AI semantic matching finds meaning beyond keywords" %}

{% trans "After filtering, the system uses semantic embeddings to compare the meaning of the query with listing descriptions. This helps connect real-world phrasing to relevant housing even when terminology differs." %}

{% trans "4) Smart re-ranking keeps results relevant and explainable" %}

{% trans "Results are re-ranked using light, explainable scoring: boosts for clear matches, gentle demotions for unrelated specialist services, and optional tuning for support level fit. Each result can show simple reasons so users understand why it appears." %}

{% trans "Privacy by design" %}

{% trans "We minimize data collection and avoid building personal profiles. Searches can be processed without storing sensitive personal details. We focus on data minimization, separation of concerns, and transparent matching — built with GDPR expectations in mind." %}

{% trans "Why this approach scales" %}

  • {% trans "Multilingual foundation: DE/FR/IT/EN from the start" %}
  • {% trans "Hybrid model: precise filtering + flexible understanding" %}
  • {% trans "Extensible data model: easy to add new support types and regions" %}
  • {% trans "Measurable improvements: ranking can be tuned using real search feedback" %}

{% trans "The result is a system that can grow from a local prototype into a broader platform — while keeping trust, safety, and clarity at the center." %}
